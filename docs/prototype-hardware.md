# プロトタイプ・ハードウェア方針

## 1. 目的

最初のプロトタイプでは、山笠上でBLE中継方式が成立するかを、不要なRF設計要因を増やさずに検証する。

検証対象は次のとおり。

- 電飾ON時のBLE検出率
- 人形の全開・全閉・途中状態による方向別遮蔽
- 前・後・左・右の4タグによる死角補完
- iPhone・Androidのバックグラウンド検出
- 必要な広告間隔と送信出力
- 外観を損なわない隠蔽位置での成立性
- 主催者端末群による位置取得・送信分散

## 2. 初期プロトタイプの基本構成

初期プロトタイプは、**一体型BLEモジュールまたは評価基板の内蔵アンテナを使用する。**

ここでいう内蔵アンテナはSoCそのものに内蔵されたアンテナではなく、BLEモジュールまたは評価基板上に実装されたPCBアンテナ、チップアンテナ、またはモジュール内蔵アンテナを指す。

```text
CR2032または評価用電源
  └── BLEモジュール／評価基板
        ├── BLE SoC
        ├── 水晶・整合回路
        ├── 内蔵PCB／チップアンテナ
        └── 書き換え可能な広告設定
```

4タグは同一ハードウェア、同一筐体、同一無線設定とし、`node_id`のみを前・後・左・右で変更する。

## 3. 一体型を優先する理由

初期検証では、方式そのものの成立性を評価する必要がある。

一体型にすることで、次の変数を除外できる。

- 同軸ケーブル損失
- RFコネクターの接触不良
- FPCアンテナの施工差
- ケーブル出口の防水
- ケーブルの振動断線
- 本体とアンテナ間の整合変化
- 年ごとのアンテナ取り回し差

問題が発生した場合に、原因を山笠の遮蔽、電飾ノイズ、スマートフォンOS制約、広告設定へ絞り込みやすくなる。

## 4. プロトタイプの段階

### 4.1 Phase 1: 市販モジュールまたは評価基板

目的はBLE方式と4方向配置の成立性確認であり、最終サイズや防水完成度の評価ではない。

必要条件：

- BLE 5.x対応
- 非接続広告が可能
- 広告間隔を変更できる
- 送信出力を変更できる
- Manufacturer Specific Data等へ独自データを設定できる
- 同一ハードウェアを4台用意できる
- 可能ならファームウェアを書き換えられる

初期設定：

| 項目 | 初期値 |
|---|---:|
| タグ数 | 4 |
| PHY | BLE 1M |
| 広告間隔 | 250ms |
| 送信出力 | 0dBm |
| 接続受付 | 無効 |
| 運用LED | 無効 |

この段階でも外観要件は免除しない。評価基板を観客から見える状態で本番相当評価してはならず、仮設であっても隠蔽候補位置へ置いて測定する。

### 4.2 Phase 2: 小型一体型専用タグ

Phase 1で成立性を確認した後、CR2032、BLEモジュールまたはSoC、アンテナ、防水筐体、固定機構を一体化する。

この段階で評価する事項：

- 実際の隠蔽空間へ収まるか
- 山笠の外観を損なわないか
- 小型化後もアンテナ性能を維持できるか
- IP67相当を満たせるか
- 1週間連続動作できるか
- 振動で電池接点が瞬断しないか

評価基板の結果を、そのまま小型製品の無線性能とみなしてはならない。基板寸法、GND面積、電池、筐体、周辺装飾によってアンテナ特性が変化するため、専用タグ状態で再試験する。

### 4.3 Phase 3: 必要箇所のみ分離アンテナ化

一体型専用タグを外観から完全に隠すと必要な無線性能を得られない方向だけ、薄型FPCアンテナ等を分離する。

分離型を採用する条件：

- 電池・基板を隠せる位置とアンテナに適した位置が一致しない
- 毎年変わる人形・装飾によって一体型アンテナが遮蔽される
- 一体型の位置変更、向き変更、送信出力調整で解決できない
- 分離後も外観を損なわない
- ケーブル長、損失、防水、振動耐性を受け入れられる

4タグすべてを同じ構造にする必要はない。実測結果により、一体型と分離型の混在を許容する。

## 5. 分離アンテナの位置付け

分離アンテナは初期プロトタイプの既定構成ではなく、外観要件と無線性能を両立できない場合の解決手段とする。

採用する場合も、山笠中央から数メートルのケーブルを引く構成にはしない。各方向のタグ本体を担当領域の近くへ隠し、短い同軸ケーブルでアンテナだけを局所的に移動する。

初期目安：

- 同軸長：10〜30cm
- 目標上限：50cm程度
- ケーブルとコネクターは筐体内でストレインリリーフ
- ケーブル出口は封止または防水構造
- FPCアンテナは非金属装飾の裏へ隠蔽

## 6. 毎年の山笠制作への対応

土台の基本形状は共通だが、人形と外側装飾は毎年変わる。

したがって、次を分離して管理する。

- **共通部分**: タグ論理仕様、ファームウェア、山笠ID、4方向の役割、試験手順
- **年次部分**: 隠蔽位置、アンテナ向き、人形可動域、LED配線、金属装飾、最終RF設定

毎年、制作途中と完成後に次の工程を行う。

1. 前・後・左・右の隠蔽候補位置を確認
2. 一体型プロトタイプを仮設
3. 人形の全開・全閉・途中状態で外観と検出率を測定
4. 電飾OFF、ON、最大輝度、演出状態で測定
5. 一体型で不足する方向を特定
6. 位置、向き、出力を調整
7. 解決しない方向だけ分離アンテナを試験
8. 合格配置を当該年度の施工記録へ保存

## 7. プロトタイプの合格条件

Phase 1またはPhase 2の一体型構成で、次を満たすか確認する。

- タグ、固定具、仮設配線が通常観覧位置から見えない
- 人形の全状態で外観を損なわない
- 電飾の見え方や動作を変えない
- 山笠周囲10m以内で20秒以内の検出成功率99%以上
- 山笠周囲20m以内で20秒以内の検出成功率95%以上
- 全方向で30秒以上の完全欠落がない
- iPhone・Androidのバックグラウンド検出が成立する

一体型が合格すれば分離アンテナは採用しない。合格しない方向だけ、原因を記録した上で分離型へ進む。

## 8. 現時点の判断

プロトタイプの既定方針は次とする。

> 市販の一体型BLEモジュールまたは評価基板で4タグ構成を作り、隠蔽配置した実際の山笠で方式を検証する。成立後に小型一体型専用タグを設計し、外観と無線性能を両立できない方向だけ分離FPCアンテナへ移行する。
