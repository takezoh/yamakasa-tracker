# ハードウェア要件

## 1. 目的

山笠に取り付けたBLEタグを、主催者アプリをインストールした周囲のスマートフォンが検出し、スマートフォン群が分担して取得した位置を観客向けWebへ公開する。

タグ自体はGPS測位やLTE通信を行わず、AirTagと同様にBLE広告のみを送信する。位置測位とインターネット通信は周囲のスマートフォンが担う。

## 2. 第一級要件：外観・意匠の保全

**ハードウェアは山笠の外観を損なってはならない。**

この要件は、無線性能、小型化、防水、保守性、取り付け容易性、コストより上位の設計制約とする。

- タグ、筐体、ブラケット、スペーサー、固定具、脱落防止具、ラベルは、通常の観覧位置から視認されてはならない
- 山笠の輪郭、造形、色、質感、電飾、人形の開閉と演出を変えてはならない
- 既存の装飾、板、布、配線の表面へ、異物として認識できる機器を露出させてはならない
- 山笠固有の意匠部品を切削、穿孔、塗装、変形してはならない
- 恒久的な接着剤、目立つ結束バンド、外部配線、外向きラベルを使用してはならない
- 取り外し後に痕跡、変色、穴、接着残りを残してはならない
- 可動人形、電飾、保守作業、安全設備へ干渉してはならない
- 外観を損なわずに無線要件を満たせない場合、外観要件を緩和せず、アンテナ、配置、タグ数またはソフトウェア側を再設計する

設計レビューと受け入れ試験には、無線・電気担当だけでなく、山笠の意匠・制作・運営責任者による外観承認を必須とする。

## 3. 対象環境

対象の山笠は、概ね次の条件を持つ。

- 高さ：約5m
- 全長：約10m
- 主構造：木材
- 内側：板や布で閉じられているが、完全密閉ではない
- 外側：全面にLEDと配線がある
- 中央下部：電飾用バッテリー
- 中央：LEDドライバー
- 四隅：大型人形が複数体あり、内側へ開閉する
- 運用：屋外、雨、水しぶき、振動、衝撃、高湿度
- 稼働期間：最大1週間
- 位置は祭り期間中、開催中・夜間停車中を含めて公開対象

## 4. システム境界

タグに必要な機能は以下に限定する。

- BLE広告送信
- 山笠IDとタグIDの送信
- 広告データの認証
- 電池状態の送信
- 1週間以上の連続運用

タグに不要な機能は以下とする。

- GPS/GNSS
- LTE、Wi-Fi、SIM
- 加速度センサー
- ジャイロ、磁気センサー
- 画面
- 常用LED
- 操作用ボタン
- イベント開催状態の判定

位置、速度、進行方向はスマートフォン群から送信された位置時系列をバックエンドで統合して算出する。タグ側の加速度を積分して進行方向を求める方式は採用しない。

## 5. 機能要件

### 5.1 BLE広告

- BLE 1M PHYを使用する
- 接続不要の非接続広告を基本とする
- 広告間隔の初期値は250msとする
- 送信出力の初期値は0dBmとし、実機試験によりタグ単位で調整可能にする
- 最大送信出力は+8dBm程度を確保する
- 4タグは同じ山笠IDを持ち、方向別のタグIDを持つ

### 5.2 識別

山笠の位置は公開情報であるため、山笠IDのローテーションは行わない。

一方、固定IDだけでは偽タグによる位置偽装が可能になるため、広告には認証コードを含める。

- `yamakasa_id`: 公開固定ID
- `node_id`: 前・後・左・右を区別する固定ID
- `boot_counter`: 電池交換や再起動を識別
- `sequence`: リプレイ検出に使う単調増加値
- `auth_tag`: 秘密鍵によるMAC

### 5.3 電源

- 電池はCR2032を1個使用する
- タグごとに独立電池とする
- 最低動作期間は1週間
- 祭り開始前に全タグの電池を新品交換する運用を基本とする
- 電池電圧低下を広告で通知する
- 降圧コンバーターやLDOは原則使用せず、SoCへ直接給電する
- BLE送信時の電圧降下対策として、低漏れのバルクコンデンサーを配置する

### 5.4 防水・耐環境

- 防水目標：IP67相当
- 雨、水しぶき、汗、泥、高湿度に耐える
- 強い振動で電池接点が瞬断しない
- 電池交換後も再密閉できる
- 基板には必要に応じてコンフォーマルコーティングを施す
- 筐体と固定部品は非導電性材料を基本とする
- 防水構造のために外観上目立つ容積、色、光沢、締結部を増やしてはならない

### 5.5 保守

- 電池交換が可能であること
- 前・後・左・右の識別は、取り外し時または保守時だけ見える位置で行う
- 製造・保守用の書き込みパッドを持つこと
- 運用中のLED点灯は行わない
- ファームウェア更新は有線書き込みを基本とし、BLE DFUは必須としない
- 点検・交換のために外観部品を損傷してはならない

## 6. 非機能要件

### 6.1 外観

外観保全を独立した合否判定項目とする。

- 昼間、夜間、電飾点灯・消灯の全状態で機器が視認されない
- 正面、後面、左右、斜め、近距離、通常観覧距離から確認する
- 写真・動画でタグや固定具が山笠の一部ではない異物として認識されない
- 筐体の色合わせだけを「非視認」とみなさず、形状、影、反射、突出も評価する
- 人形の全開・全閉・途中状態で機器が露出しない
- 外観責任者が承認しない配置は、無線性能を満たしても不採用とする

### 6.2 サイズ

外形目標は固定値ではなく、隠蔽可能な空間から逆算する。初期上限の目安は以下とする。

- 外形：30〜45mm程度
- 厚さ：10〜15mm程度
- 重量：10〜20g程度

防水やアンテナ効率のために外観を犠牲にしてはならない。隠蔽空間へ収まらない場合は、部品、基板、電池ホルダー、アンテナ形式を再設計する。

### 6.3 無線性能

山笠周辺の主催者スマートフォンが、どの方向からでも少なくとも1タグを安定して検出できること。

目標値：

- 山笠周囲10m以内：20秒以内の検出成功率99%以上
- 山笠周囲20m以内：20秒以内の検出成功率95%以上
- 全方向で30秒以上の完全欠落が発生しない

RSSIの絶対値ではなく、所定時間内の検出成功率を主要指標とする。ただし、無線性能を満たすためにタグを外観上露出させることは認めない。

### 6.4 セキュリティ

- タグごとに秘密鍵を持つ
- 秘密鍵は広告しない
- 広告データにはMACを付与する
- リプレイを検出できること
- 公開IDの秘匿は目的としない

## 7. 推奨部品クラス

| 部品 | 要件 |
|---|---|
| BLE SoC | +8dBm程度に対応する低消費電力SoC。例：Nordic nRF52833相当 |
| 電池 | CR2032 |
| 電池ホルダー | 薄型かつ振動に強いロック式、または筐体で押圧する構造 |
| アンテナ | PCBまたはFPC。隠蔽状態を含む実装状態で調整可能 |
| 水晶 | 32MHz必須。低速クロックは精度要件に応じて搭載 |
| 筐体 | 低光沢・非導電性・IP67相当。外観から見えない形状と取り付けを前提とする |
| 固定具 | 既存構造内に隠蔽でき、無痕で撤去可能な非導電性固定具 |

## 8. 受け入れ条件

- 4タグすべてが新品CR2032で168時間以上連続動作する
- 電飾最大点灯中でも無線性能目標を満たす
- 人形の全開・全閉・一部開閉状態で無線性能目標を満たす
- 雨天相当の散水後も動作する
- 振動試験後に電池瞬断、筐体破損、浸水がない
- 4タグ中1タグが停止しても、周囲の主要方向で位置更新を継続できる
- 昼夜・全方向・人形全状態でタグ、筐体、固定具、ラベルが観覧者から視認されない
- 山笠の輪郭、造形、電飾、可動、色、質感を変えていない
- 取り付けと撤去で山笠に不可逆な変更や痕跡を残さない
- 山笠の意匠・制作・運営責任者による外観承認を得る

外観要件と無線要件のどちらか一方でも満たさない構成は受け入れない。
