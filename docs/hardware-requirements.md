# ハードウェア要件

## 1. 目的

山笠に取り付けたBLEタグを、主催者アプリをインストールした周囲のスマートフォンが検出し、端末群が分担して取得した位置を観客向けWebへ公開する。

タグ自身はGPS測位やLTE通信を行わず、BLE広告のみを送信する。位置測位とインターネット通信は周囲のスマートフォンが担う。

## 2. 第一級要件：外観・意匠の保全

**ハードウェアは山笠の外観を損なってはならない。**

この要件は、無線性能、小型化、防水、保守性、取り付け容易性、コストより上位の設計制約とする。

- タグ、筐体、アンテナ、固定具、脱落防止具、ラベルは通常の観覧位置から視認されない
- 山笠の輪郭、造形、色、質感、電飾、人形の開閉と演出を変えない
- 外装表面へ異物として認識できる機器を露出させない
- 意匠部品を切削、穿孔、塗装、変形しない
- 取り外し後に穴、変色、接着残りなどの痕跡を残さない
- 可動人形、電飾、保守作業、安全設備へ干渉しない
- 外観を保ったまま無線要件を満たせない場合、外観要件を緩和せず、アンテナ、配置、タグ数、筐体またはソフトウェアを再設計する

無線・電気担当だけでなく、山笠の意匠・制作・運営責任者による外観承認を受け入れ条件とする。

## 3. 対象環境

- 高さ：約5m
- 全長：約10m
- 主構造：木材
- 内側：板や布で閉じられているが完全密閉ではない
- 外側：全面にLEDと配線がある
- 中央下部：電飾用バッテリー
- 中央：LEDドライバー
- 四隅：大型人形が複数体あり、内側へ開閉する
- 運用：屋外、雨、水しぶき、振動、衝撃、高湿度
- 稼働期間：最大1週間
- 祭り期間中は開催中・夜間停車中を含めて位置を公開する
- 土台の基本形状は毎年共通するが、人形と外側の装飾は毎年変わる

## 4. システム境界

タグに必要な機能：

- BLE広告送信
- `resource_id`と`node_id`の送信
- 広告データの認証
- 電池状態の送信
- 168時間以上の連続運用

タグに搭載しない機能：

- GPS/GNSS
- LTE、Wi-Fi、SIM
- 加速度センサー、ジャイロ、磁気センサー
- 画面、常用LED、操作ボタン
- 開催状態判定

位置、速度、進行方向はスマートフォン群から送信された位置時系列をバックエンドで統合して算出する。

## 5. 識別子

### 5.1 統一概念

追跡対象の識別子は、BLE広告、モバイルアプリ、API、データベース、ログの全層で **`resource_id`** に統一する。

`resource_id`は山笠だけに限定しない一般概念であり、将来ほかの山車や移動イベント対象にも利用できる。

### 5.2 表現

- 論理型：符号なし64-bit ID
- BLE広告：8-byte big-endian binary
- API・JSON・ログ：16桁の小文字16進文字列
- DB：`BIGINT`相当または8-byte binary

例：

```text
BLE:  01 af 34 c9 81 2d e7 04
API:  "01af34c9812de704"
```

概念名と値は全層で同一とし、`yamakasa_id`、`beacon_id`、`resource_ref`などの別名や変換用識別子を設けない。

### 5.3 ノード識別子

`node_id`は同じ追跡対象に属する4タグを識別する。

- `front`
- `rear`
- `left`
- `right`

4タグは同じ`resource_id`を持ち、異なる`node_id`を持つ。

### 5.4 公開と認証

位置と`resource_id`は公開情報であるため、IDローテーションは行わない。ただし、偽タグとリプレイを防ぐため広告へ認証コードを含める。

- `resource_id`
- `node_id`
- `boot_counter`
- `sequence`
- `battery_state`
- `auth_tag`

## 6. BLE広告

- BLE 1M PHY
- 接続不要の非接続広告
- 広告間隔の初期値：250ms
- 送信出力の初期値：0dBm
- 実機試験によりタグ単位で-4、0、+4、+8dBmから選択
- 最大送信出力：+8dBm程度
- 運用中のLED点灯なし

## 7. 電源

- タグごとにCR2032を1個使用
- 最低動作期間：168時間
- 祭り開始前に全タグの電池を新品交換
- 電池状態を広告で通知
- LDOや降圧コンバーターは原則使用せずSoCへ直接給電
- BLE送信時の電圧降下対策として低漏れバルクコンデンサーを配置
- 振動で電池接点が瞬断しない保持構造

## 8. 防水・耐環境

- 防水目標：IP67相当
- 雨、水しぶき、汗、泥、高湿度へ耐える
- 電池交換後も再密閉できる
- 必要に応じて基板へコンフォーマルコーティング
- 筐体と固定部品は非導電性材料を基本とする
- 防水構造のために外観上目立つ容積、光沢、締結部を増やさない

## 9. 形状・アンテナ

外形は固定値から決めず、その年の山笠で利用できる隠蔽空間から逆算する。初期上限目安：

- 外形：30〜45mm
- 厚さ：10〜15mm
- 重量：10〜20g

プロトタイプは一体型BLEモジュールの内蔵アンテナを使用する。製品化も小型一体型を基本とし、外観と無線性能を両立できない方向だけ短い同軸ケーブル付きFPCアンテナ分離型を採用する。

## 10. 無線性能

- 山笠周囲10m以内：20秒以内の検出成功率99%以上
- 山笠周囲20m以内：20秒以内の検出成功率95%以上
- 全方向で30秒以上の完全欠落なし

RSSIの絶対値ではなく、所定時間内の検出成功率を主要指標とする。無線性能のためにタグを露出させてはならない。

## 11. 受け入れ条件

- 4タグすべてが新品CR2032で168時間以上連続動作する
- 電飾最大点灯中でも無線性能目標を満たす
- 人形の全開・全閉・途中状態で無線性能目標を満たす
- 雨天相当の散水後も動作する
- 振動試験後に電池瞬断、筐体破損、浸水がない
- 4タグ中1タグが停止しても主要方向で位置更新を継続できる
- 昼夜・全観覧方向・人形の全状態で機器が視認されない
- 制作・意匠・運営責任者が外観を承認する
